# 4월 검색·추천 데이터 구조 개선 정리

## 1. 작업 목적

4월에는 프로젝트 메타데이터 저장 구조, AI 메타데이터 연동 구조, 검색 인덱싱 데이터 구조, 검색 API 입출력 흐름을 중심으로 백엔드 데이터 흐름을 검토하고 일부 구조를 보완하였다.

## 2. 주요 개선 내용

### 2.1 ProjectIndexDocument domain 필드 추가

기존 Project 엔티티에는 domain 정보가 존재하였으나, 검색 인덱싱 문서인 ProjectIndexDocument에는 domain 정보가 포함되어 있지 않았다. 이에 따라 검색 및 추천 기능에서 프로젝트 분야 정보를 활용할 수 있도록 ProjectIndexDocument에 domain 필드를 추가하였다.

### 2.2 인덱싱 데이터 매핑 보완

ProjectService의 publishIndexEvent 메서드에서 Project 엔티티의 domain 값을 ProjectIndexDocument로 전달하도록 수정하였다. 이를 통해 프로젝트 등록 또는 수정 이후 검색 인덱싱 이벤트가 발생할 때 프로젝트 분야 정보가 함께 전달될 수 있도록 하였다.

### 2.3 벡터 검색 메타데이터 보완

ProjectIndexEventListener에서 벡터 검색 저장소로 전달하는 metadata에 domain 항목을 추가하였다. 이를 통해 향후 임베딩 기반 추천 또는 벡터 검색 결과에서 프로젝트 분야 정보를 함께 활용할 수 있도록 하였다.

### 2.4 검색 요청 및 결과 구조 보완

SearchRequest와 SearchQuery에 domain 조건을 추가하여 프로젝트 분야 기반 검색 요청이 가능하도록 하였다. 또한 ProjectSearchResult와 ProjectResponse의 검색 결과 매핑 구조를 수정하여 검색 API 응답에서도 domain 값이 누락되지 않도록 보완하였다.

### 2.5 Mock 검색 어댑터 필터링 로직 보완

MockProjectSearchAdapter에서 domain 필터링 로직을 추가하고, 키워드 검색 대상에도 domain을 포함하였다. 또한 문자열 비교 시 null 값으로 인해 오류가 발생할 가능성을 줄이기 위해 containsIgnoreCase 메서드를 활용하였다.

## 3. 검증 내용

수정 후 compileJava를 실행하여 전체 백엔드 코드가 정상적으로 컴파일되는 것을 확인하였다. 또한 프로젝트 등록 및 검색 API 흐름을 통해 domain 값이 요청 조건, 검색 조건, 검색 결과, API 응답 구조에서 일관되게 활용될 수 있는지 확인하였다.

테스트용 프로젝트를 서로 다른 domain 값으로 등록한 뒤, domain 조건을 포함한 검색 API를 호출하였다. 그 결과 `교내 프로젝트 아카이브`와 `산업 안전` domain 조건에서 각각 해당 프로젝트만 조회되는 것을 확인하였다.

## 4. 5월 보완 사항

- 실제 OpenSearch 연동 어댑터 구현
- ProjectIndexDocument와 OpenSearch 인덱스 매핑 구조 구체화
- 임베딩 벡터 저장 방식 및 벡터 검색 시스템 연동 방식 검토
- 프로젝트 등록/수정 이후 검색 인덱스 자동 동기화 흐름 테스트
- 검색 조건별 API 테스트 케이스 정리
- AI 추천 API 응답 구조와 검색 인덱싱 데이터 연결 방식 보완

## 5. 4월 최종 정리

4월에는 프로젝트 메타데이터 저장 흐름을 실제 API로 검증하고, AI 메타데이터 및 검색 인덱싱 데이터 구조를 백엔드 코드와 연결하는 작업을 진행하였다. 특히 프로젝트 분야 정보인 domain이 검색 인덱싱 문서, 검색 요청, 검색 결과, API 응답 구조에서 일관되게 활용되도록 관련 코드를 보완하였다.

또한 프로젝트 수정·삭제 기능에서 반복되던 사용자 조회 및 작성자·관리자 권한 검증 로직을 공통 메서드로 분리하여 권한 검증 흐름의 중복을 줄였다. 이를 통해 프로젝트 데이터 관리 기능과 검색·추천 데이터 흐름이 보다 일관되게 유지될 수 있도록 개선하였다.

## 6. 5월 개발 계획

6월 최종 완성을 고려하여 5월에는 실제 구현과 통합 테스트 중심으로 진행할 예정이다.

- 실제 OpenSearch 연동 어댑터 구현
- 프로젝트 등록·수정·삭제 시 검색 인덱스 자동 동기화 처리
- ProjectIndexDocument와 OpenSearch 인덱스 필드 매핑 구조 확정
- AI 모듈에서 생성한 요약, 기술스택, 도메인, 난이도 결과를 프로젝트 데이터와 연결
- 임베딩 벡터 저장 및 벡터 검색 시스템 연동 방식 구현
- 자연어 질의 기반 AI 추천 API가 실제 검색·추천 결과를 반환하도록 보완
- 프론트엔드와 백엔드 API 연동 테스트
- 6월 최종 발표를 대비한 시연용 데이터 정리 및 오류 수정
